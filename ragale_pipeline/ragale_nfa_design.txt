================================================================================
  NFA INPUT ALPHABET
================================================================================

  ОЈ = { (w, c) | w вҲҲ {I, U}, c вҲҲ CONSONANTS вҲӘ {вҲ…} }  вҲӘ  { \n }

  Each symbol is a syllable token from the FST:
    w = weight (I = Laghu, U = Guru)
    c = base consonant of the syllable (вҲ… if vowel-initial)
    \n = line separator

  Shorthand in diagrams:
    I_c  = (I, c)    any laghu with consonant c
    U_c  = (U, c)    any guru with consonant c
    I_*  = (I, any)   laghu with any consonant
    U_*  = (U, any)   guru with any consonant
    *_c  = (any, c)   any weight with specific consonant c

================================================================================
  ДҖDI PRДҖSA вҖ” CONSONANT MATCHING MECHANISM
================================================================================

  The prДҒsa check is encoded by "parameterizing" the NFA on consonant c.

  Conceptually, there are |C| copies of the line-2 NFA, one for each
  consonant. The Оө-branch from START non-deterministically picks which
  consonant the 2nd syllable will have, then the NFA for that consonant
  only accepts if both lines actually have that consonant at position 2.
