================================================================================
  1.  PRELIMINARIES: FORMAL LANGUAGE DEFINITIONS
================================================================================

Definition 1 (Alphabet).
    An alphabet ОЈ is a finite, non-empty set of symbols.

Definition 2 (String).
    A string w over ОЈ is a finite sequence w = aвӮҒaвӮӮ...aвӮҷ where aбөў вҲҲ ОЈ.
    The empty string is denoted Оө. The set of all strings over ОЈ is ОЈ*.

Definition 3 (Language).
    A language L is a subset of ОЈ*. A language is regular if it can be
    recognized by a finite automaton (DFA or NFA).

Definition 4 (Context-Free Grammar).
    A CFG G = (V, ОЈ, P, S) where:
    - V is a finite set of non-terminal symbols
    - ОЈ is a finite set of terminal symbols (V вҲ© ОЈ = вҲ…)
    - P is a finite set of production rules V вҶ’ (V вҲӘ ОЈ)*
    - S вҲҲ V is the start symbol

Definition 5 (Right-Linear Grammar).
    A CFG where every production has the form:
        A вҶ’ wB    (w вҲҲ ОЈ*, B вҲҲ V)
        A вҶ’ w     (w вҲҲ ОЈ*)
    Right-linear grammars generate exactly the regular languages.

Theorem 1 (Equivalence).
    The following are equivalent for a language L:
    (a) L is recognized by a DFA
    (b) L is recognized by an NFA
    (c) L is generated by a right-linear grammar
    (d) L is described by a regular expression

================================================================================
  2.  FINITE STATE TRANSDUCERS (FST) вҖ” THEORY
================================================================================

Definition 6 (Finite State Transducer).
    An FST T = (Q, ОЈ, О“, Оҙ, qвӮҖ, F) where:
    - Q  = finite set of states
    - ОЈ  = input alphabet
    - О“  = output alphabet
    - Оҙ  = transition function: Q Г— (ОЈ вҲӘ {Оө}) вҶ’ P(Q Г— О“*)
    - qвӮҖ вҲҲ Q is the initial state
    - F вҠҶ Q is the set of accepting states

    An FST reads input symbols and produces output symbols. It is a
    transduction from ОЈ* to О“* вҖ” i.e., a function (or relation) that
    maps input strings to output strings.

Definition 7 (Deterministic FST).
    An FST where for every state q and input symbol a, there is exactly
    one transition: |Оҙ(q, a)| = 1 for all q вҲҲ Q, a вҲҲ ОЈ.

Observation (Pipeline).
    FSTs compose: if TвӮҒ: ОЈвӮҒ* вҶ’ ОЈвӮӮ* and TвӮӮ: ОЈвӮӮ* вҶ’ ОЈвӮғ*, then
    TвӮӮ вҲҳ TвӮҒ: ОЈвӮҒ* вҶ’ ОЈвӮғ* is also realizable as an FST. This is the basis
    for chaining our pipeline stages.

================================================================================
  3.  NON-DETERMINISTIC FINITE AUTOMATA (NFA) вҖ” THEORY
================================================================================

Definition 8 (NFA).
    An NFA M = (Q, ОЈ, Оҙ, qвӮҖ, F) where:
    - Q  = finite set of states
    - ОЈ  = input alphabet
    - Оҙ  = transition function: Q Г— (ОЈ вҲӘ {Оө}) вҶ’ P(Q)
    - qвӮҖ вҲҲ Q is the initial state
    - F вҠҶ Q is the set of accepting states

    A string w is accepted if there EXISTS a path from qвӮҖ to some
    state in F labeled by w (including Оө-transitions).

Definition 9 (Branch Set).
    At any point during NFA execution on input wвӮҒwвӮӮ...wвӮ–, the active
    state is a set S вҠҶ Q of states reachable from qвӮҖ via any path
    labeled by wвӮҒwвӮӮ...wвӮ–. The string is accepted if S вҲ© F вү  вҲ… after
    processing the entire input.

Implementation Note.
    Our GanaNFA maintains the branch set explicitly. Each branch is a
    tuple (slot, gana_name, sub_pos, matched_ganas). The _advance()
    function computes the next branch set given a symbol.

================================================================================
  4.  RIGHT-LINEAR GRAMMARS AND THE CHOMSKY HIERARCHY
================================================================================

Chomsky Hierarchy:
    Type 3 (Regular)        вҠӮ  Type 2 (Context-Free)
                            вҠӮ  Type 1 (Context-Sensitive)
                            вҠӮ  Type 0 (Recursively Enumerable)

Regular languages are exactly those recognized by DFA/NFA, and exactly
those generated by right-linear (or left-linear) grammars. Since all our
constraints are regular, we can express the entire Ragale validation as
a right-linear CFG.

Key Insight:
    Every NFA state q becomes a non-terminal in the right-linear grammar.
    Every transition Оҙ(q, a) вҲӢ q' becomes a production:  q вҶ’ a q'
    Every accepting state q вҲҲ F gets an additional production: q вҶ’ Оө

This mechanical correspondence is the basis for Sections 5-9 below.

  5.  STAGE 1 FST: SYLLABLE ASSEMBLER вҖ” FORMAL SPECIFICATION & CFG
================================================================================

PURPOSE: Transform Kannada Unicode character stream вҶ’ syllable stream.

FORMAL DEFINITION:
    TвӮҒ = (QвӮҒ, ОЈ_char, О“_syl, ОҙвӮҒ, IDLE, {IDLE})

    QвӮҒ = { IDLE, CONS, PEND_V, VOWEL }

    ОЈ_char = C вҲӘ V вҲӘ M вҲӘ {H} вҲӘ D вҲӘ {SP, NL, SKIP, OTHER}
        where:
        C  = {аІ•, аІ–, аІ—, аІҳ, аІҷ, аІҡ, аІӣ, аІң, аІқ, аІһ, аІҹ, аІ , аІЎ, аІў, аІЈ,
              аІӨ, аІҘ, аІҰ, аІ§, аІЁ, аІӘ, аІ«, аІ¬, аІӯ, аІ®, аІҜ, аІ°, аІІ, аІө,
              аІ¶, аІ·, аІё, аІ№, аІі}                              (34 consonants)
        V  = {аІ…, аІҶ, аІҮ, аІҲ, аІү, аІҠ, аІӢ, аІҺ, аІҸ, аІҗ, аІ’, аІ“, аІ”}     (13 vowels)
        M  = {аІҫ, аІҝ, аіҖ, аіҒ, аіӮ, аіғ, аіҶ, аіҮ, аіҲ, аіҠ, аіӢ, аіҢ}        (12 matras)
        H  = {аіҚ}                                            (halant/virama)
        D  = {аІӮ, аІғ}                                         (diacritics)

    О“_syl = syllable strings вҲӘ {SP, NL}

RIGHT-LINEAR GRAMMAR GвӮҒ:
    (Non-terminals = FST states; terminals = character categories)

    IDLE   вҶ’ c  CONS            for c вҲҲ C
    IDLE   вҶ’ v  VOWEL           for v вҲҲ V
    IDLE   вҶ’ SP IDLE            (emit boundary)
    IDLE   вҶ’ NL IDLE            (emit boundary)
    IDLE   вҶ’ Оө                  (accept: end of input)

    CONS   вҶ’ H  PEND_V          (virama вҶ’ pending)
    CONS   вҶ’ m  IDLE            for m вҲҲ M вҲӘ D  (matra/diacritic вҶ’ emit syllable)
    CONS   вҶ’ c  CONS            for c вҲҲ C  (new consonant вҶ’ emit prev, start new)
    CONS   вҶ’ v  VOWEL           for v вҲҲ V  (vowel вҶ’ emit prev, start vowel)
    CONS   вҶ’ SP IDLE            (emit syllable + boundary)
    CONS   вҶ’ NL IDLE            (emit syllable + boundary)

    PEND_V вҶ’ c  CONS            for c вҲҲ C  (conjunct: CаұҚC continues cluster)
    PEND_V вҶ’ m  IDLE            for m вҲҲ M вҲӘ D  (unusual: emit syllable)
    PEND_V вҶ’ SP IDLE            (pollu merge + boundary)
    PEND_V вҶ’ NL IDLE            (pollu merge + boundary)
    PEND_V вҶ’ v  VOWEL           for v вҲҲ V  (pollu merge, start vowel)

    VOWEL  вҶ’ d  IDLE            for d вҲҲ D  (diacritic вҶ’ emit vowel+diac)
    VOWEL  вҶ’ c  CONS            for c вҲҲ C  (emit vowel, start consonant)
    VOWEL  вҶ’ v  VOWEL           for v вҲҲ V  (emit vowel, start new vowel)
    VOWEL  вҶ’ SP IDLE            (emit vowel + boundary)
    VOWEL  вҶ’ NL IDLE            (emit vowel + boundary)

  TRANSDUCTION OUTPUT:
    Each production that transitions to IDLE (or from CONS/VOWEL to a new
    start) implicitly emits the accumulated buffer as a syllable token.

  6.  STAGE 2 FST: GURU/LAGHU CLASSIFIER вҖ” FORMAL SPECIFICATION & CFG
================================================================================

PURPOSE: Transform syllable stream вҶ’ (syllable, I/U) stream.

FORMAL DEFINITION:
    TвӮӮ = (QвӮӮ, О“_syl, О“_IU, ОҙвӮӮ, EMPTY, {EMPTY})

    QвӮӮ = { EMPTY, PEND_I }

    О“_syl = { syllables, boundaries }
    О“_IU  = { (syl, "I"), (syl, "U") }

    Classification function (intrinsic):
        guru(s) = true  iff  s has long matra вҲҲ {аІҫ, аіҖ, аіӮ, аіҮ, аіӢ, аіҢ}
                         or  s вҲҲ {аІҶ, аІҲ, аІҠ, аІҸ, аІ“}
                         or  s contains аІҗ, аІ”, аіҲ, or аіҢ
                         or  s contains аІӮ or аІғ
                         or  s ends with аіҚ

    Conjunct trigger function:
        conj(s) = true  iff  вҲғi: s[i] вҲҲ C вҲ§ s[i+1] = аіҚ вҲ§ s[i+2] вҲҲ C

RIGHT-LINEAR GRAMMAR GвӮӮ:
    (Syllables are partitioned into classes: S_U = guru syllables,
     S_I = laghu syllables, S_Ic = laghu before conjunct trigger,
     S_In = laghu before non-conjunct)

    EMPTY  вҶ’ s_U  EMPTY           for s with guru(s) = true   (emit U)
    EMPTY  вҶ’ s_I  PEND_I          for s with guru(s) = false  (buffer)
    EMPTY  вҶ’ Оө                    (accept)

    PEND_I вҶ’ s_Ic EMPTY           (emit buffered as U, process s)   Rule 5
    PEND_I вҶ’ s_Ic PEND_I          (emit buffered as U, buffer s)    Rule 5
    PEND_I вҶ’ s_In EMPTY           (emit buffered as I, process s)
    PEND_I вҶ’ s_In PEND_I          (emit buffered as I, buffer s)
    PEND_I вҶ’ s_U  EMPTY           (emit buffered as ?, emit s as U)
    PEND_I вҶ’ BOUND EMPTY          (emit buffered as I, boundary)

  Where:
    s_Ic = syllable where conj(s) = true  (triggers Rule 5 on predecessor)
    s_In = syllable where conj(s) = false (no Rule 5)
    s_U  = syllable where guru(s) = true
    BOUND = space or newline

  TRANSDUCTION OUTPUT:
    Each production emits a (syllable, "I" or "U") tuple for the
    resolved syllable.

  7.  STAGE 3 NFA: GANA PATTERN VALIDATOR вҖ” FORMAL SPECIFICATION & CFG
================================================================================

PURPOSE: Validate that a sequence of I/U markers forms a valid Ragale
         gana partition: (III|IIU)Ві В· IIU = exactly 12 symbols.

FORMAL DEFINITION:
    MвӮғ = (QвӮғ, {I, U}, ОҙвӮғ, SвӮҖ, {ACCEPT})

    The NFA has states encoding (slot, gana_pattern, position_within_gana):

    QвӮғ = { SвӮҖ,                                            (start)
            G1_III_0, G1_III_1, G1_III_2,                  (gana 1, III)
            G1_IIU_0, G1_IIU_1, G1_IIU_2,                 (gana 1, IIU)
            G2_III_0, G2_III_1, G2_III_2,                  (gana 2, III)
            G2_IIU_0, G2_IIU_1, G2_IIU_2,                 (gana 2, IIU)
            G3_III_0, G3_III_1, G3_III_2,                  (gana 3, III)
            G3_IIU_0, G3_IIU_1, G3_IIU_2,                 (gana 3, IIU)
            G4_IIU_0, G4_IIU_1, G4_IIU_2,                 (gana 4, IIU only)
            ACCEPT }

    |QвӮғ| = 1 + 6 + 6 + 6 + 3 + 1 = 23 states

TRANSITION TABLE:
    (Оө-transitions from start and gana completions shown as вҶ’ОөвҶ’)

    SвӮҖ вҶ’ОөвҶ’ G1_III_0
    SвӮҖ вҶ’ОөвҶ’ G1_IIU_0

    в”Җв”Җв”Җ GANA 1 (slot 0): III or IIU в”Җв”Җв”Җ

    G1_III_0 в”ҖIвҶ’ G1_III_1
    G1_III_1 в”ҖIвҶ’ G1_III_2
    G1_III_2 в”ҖIвҶ’ {G2_III_0, G2_IIU_0}         (gana 1 complete вҶ’ spawn gana 2)

    G1_IIU_0 в”ҖIвҶ’ G1_IIU_1
    G1_IIU_1 в”ҖIвҶ’ G1_IIU_2
    G1_IIU_2 в”ҖUвҶ’ {G2_III_0, G2_IIU_0}         (gana 1 complete вҶ’ spawn gana 2)

    в”Җв”Җв”Җ GANA 2 (slot 1): III or IIU в”Җв”Җв”Җ

    G2_III_0 в”ҖIвҶ’ G2_III_1
    G2_III_1 в”ҖIвҶ’ G2_III_2
    G2_III_2 в”ҖIвҶ’ {G3_III_0, G3_IIU_0}         (gana 2 complete вҶ’ spawn gana 3)

    G2_IIU_0 в”ҖIвҶ’ G2_IIU_1
    G2_IIU_1 в”ҖIвҶ’ G2_IIU_2
    G2_IIU_2 в”ҖUвҶ’ {G3_III_0, G3_IIU_0}         (gana 3 complete вҶ’ spawn gana 3)

    в”Җв”Җв”Җ GANA 3 (slot 2): III or IIU в”Җв”Җв”Җ

    G3_III_0 в”ҖIвҶ’ G3_III_1
    G3_III_1 в”ҖIвҶ’ G3_III_2
    G3_III_2 в”ҖIвҶ’ G4_IIU_0                     (gana 3 complete вҶ’ gana 4, IIU only)

    G3_IIU_0 в”ҖIвҶ’ G3_IIU_1
    G3_IIU_1 в”ҖIвҶ’ G3_IIU_2
    G3_IIU_2 в”ҖUвҶ’ G4_IIU_0                     (gana 3 complete вҶ’ gana 4, IIU only)

    в”Җв”Җв”Җ GANA 4 (slot 3): IIU ONLY (enforces Guru ending) в”Җв”Җв”Җ

    G4_IIU_0 в”ҖIвҶ’ G4_IIU_1
    G4_IIU_1 в”ҖIвҶ’ G4_IIU_2
    G4_IIU_2 в”ҖUвҶ’ ACCEPT                       (line complete!)

RIGHT-LINEAR GRAMMAR GвӮғ:
    (Each NFA state becomes a non-terminal)

    SвӮҖ       вҶ’ I G1_III_1                     (start gana 1 as III)
    SвӮҖ       вҶ’ I G1_IIU_1                     (start gana 1 as IIU)

    в”Җв”Җв”Җ Gana 1 paths в”Җв”Җв”Җ

    G1_III_1  вҶ’ I G1_III_2
    G1_III_2  вҶ’ I G2_III_1                    (III complete, gana 2 starts as III)
    G1_III_2  вҶ’ I G2_IIU_1                    (III complete, gana 2 starts as IIU)

    G1_IIU_1  вҶ’ I G1_IIU_2
    G1_IIU_2  вҶ’ U G2_III_1                    (IIU complete, gana 2 as III)
    G1_IIU_2  вҶ’ U G2_IIU_1                    (IIU complete, gana 2 as IIU)

    в”Җв”Җв”Җ Gana 2 paths в”Җв”Җв”Җ

    G2_III_1  вҶ’ I G2_III_2
    G2_III_2  вҶ’ I G3_III_1                    (III complete, gana 3 as III)
    G2_III_2  вҶ’ I G3_IIU_1                    (III complete, gana 3 as IIU)

    G2_IIU_1  вҶ’ I G2_IIU_2
    G2_IIU_2  вҶ’ U G3_III_1                    (IIU complete, gana 3 as III)
    G2_IIU_2  вҶ’ U G3_IIU_1                    (IIU complete, gana 3 as IIU)

    в”Җв”Җв”Җ Gana 3 paths в”Җв”Җв”Җ

    G3_III_1  вҶ’ I G3_III_2
    G3_III_2  вҶ’ I G4_IIU_1                    (III complete вҶ’ forced IIU for gana 4)

    G3_IIU_1  вҶ’ I G3_IIU_2
    G3_IIU_2  вҶ’ U G4_IIU_1                    (IIU complete вҶ’ forced IIU for gana 4)

    в”Җв”Җв”Җ Gana 4 path (IIU only) в”Җв”Җв”Җ

    G4_IIU_1  вҶ’ I G4_IIU_2
    G4_IIU_2  вҶ’ U                              (ACCEPT вҖ” terminal production)

NOTE: The first symbol of each gana is always I (since both III and IIU
start with I). The Оө-transitions from SвӮҖ and gana completions are
absorbed by combining: SвӮҖ вҶ’ОөвҶ’ G1_III_0 в”ҖIвҶ’ G1_III_1 becomes SвӮҖ вҶ’ I G1_III_1.

LANGUAGE GENERATED:
    L(GвӮғ) = { w вҲҲ {I,U}В№ВІ : w = gвӮҒgвӮӮgвӮғgвӮ„ where
               gбөў вҲҲ {III, IIU} for i вҲҲ {1,2,3} and gвӮ„ = IIU }

    Cardinality: |L(GвӮғ)| = 2Ві Г— 1 = 8 valid patterns per line.

ENUMERATION OF ALL 8 VALID LINE PATTERNS:
    1.  I I I  I I I  I I I  I I U      (III III III IIU)
    2.  I I I  I I I  I I U  I I U      (III III IIU IIU)
    3.  I I I  I I U  I I I  I I U      (III IIU III IIU)
    4.  I I I  I I U  I I U  I I U      (III IIU IIU IIU)
    5.  I I U  I I I  I I I  I I U      (IIU III III IIU)
    6.  I I U  I I I  I I U  I I U      (IIU III IIU IIU)
    7.  I I U  I I U  I I I  I I U      (IIU IIU III IIU)
    8.  I I U  I I U  I I U  I I U      (IIU IIU IIU IIU)

REGULAR EXPRESSION:
    L(GвӮғ) = (III | IIU)(III | IIU)(III | IIU)(IIU)
           = II(I|U) II(I|U) II(I|U) IIU

  8.  STAGE 4 NFA: ДҖDI PRДҖSA VALIDATOR вҖ” FORMAL SPECIFICATION & CFG
================================================================================

PURPOSE: Validate that the 2nd syllable's base consonant matches between
         both lines of the couplet.

FORMAL DEFINITION:
    MвӮ„ = (QвӮ„, ОЈ_syl, ОҙвӮ„, L1_S0, {ACC})

    The input alphabet is the set of all possible syllables, partitioned
    by their base consonant:
        ОЈ_syl = {s_c : c вҲҲ C вҲӘ {вҲ…}}
    where s_c denotes any syllable whose base consonant is c.

    QвӮ„ = { L1_S0, L1_S1, L1_REST, NL,
            L2_S0, L2_S1_c (for each c вҲҲ C вҲӘ {вҲ…}),
            ACC, REJ }

    For exact match only (strict=True):

    L1_S0  в”Җв”Җs_*в”Җв”ҖвҶ’ L1_S1                  (consume 1st syllable, any consonant)
    L1_S1  в”Җв”Җs_cв”Җв”ҖвҶ’ L1_REST_c              (store consonant c from 2nd syllable)
                    (for each consonant c)
    L1_REST_c в”Җв”Җs_*в”Җв”ҖвҶ’ L1_REST_c           (consume rest of line 1)
    L1_REST_c в”Җв”ҖNLв”Җв”ҖвҶ’  L2_S0_c             (newline вҶ’ start line 2)
    L2_S0_c  в”Җв”Җs_*в”Җв”ҖвҶ’ L2_S1_c             (consume 1st syllable of line 2)
    L2_S1_c  в”Җв”Җs_cв”Җв”ҖвҶ’ ACC                  (2nd syl matches вҶ’ accept!)
    L2_S1_c  в”Җв”Җs_dв”Җв”ҖвҶ’ REJ                  (d вү  c вҶ’ reject)

    With equivalence groups (strict=False), the match at L2_S1 checks:
        c вүЎ d  iff  c = d  or  {c,d} вҠҶ {аІІ,аІі}  or  {c,d} вҠҶ {аІ¶,аІ·,аІё}

RIGHT-LINEAR GRAMMAR GвӮ„ (for a single consonant c):
    L1_S0     вҶ’ s L1_S1                    (any syllable)
    L1_S1     вҶ’ s_c L1_REST                (2nd syllable has consonant c)
    L1_REST   вҶ’ s L1_REST                  (consume rest of line 1)
    L1_REST   вҶ’ NL L2_S0                   (newline)
    L2_S0     вҶ’ s L2_S1                    (1st syllable of line 2)
    L2_S1     вҶ’ s_c                         (2nd syllable matches вҶ’ accept)

    The full grammar is the union over all consonants c:
    GвӮ„ = вҲӘ_{c вҲҲ C} GвӮ„(c)

    This union captures the non-deterministic "guess" of which consonant
    the prasa will use вҖ” exactly as described in the NFA design document's
    36-copy parallel architecture.

================================================================================
  9.  COMBINED: TWO-LINE RAGALE VALIDATOR вҖ” FULL CFG
================================================================================

The complete Ragale validator combines all stages into a single language.
Since each stage is regular, their intersection is also regular (regular
languages are closed under intersection), and expressible as a
right-linear grammar.

COMBINED LANGUAGE (informal):
    L_ragale = L_gana_line1 вҲ© L_gana_line2 вҲ© L_prasa вҲ© L_guru_ending

Where:
    L_gana_line = { w вҲҲ {I,U}В№ВІ : w matches (III|IIU)Ві В· IIU }
    L_prasa     = { (lвӮҒ, lвӮӮ) : base_cons(sylвӮӮ(lвӮҒ)) вүЎ base_cons(sylвӮӮ(lвӮӮ)) }
    L_guru_end  = { w : w[12] = U }   (already enforced by gana 4 = IIU)

FULL RIGHT-LINEAR GRAMMAR G_ragale:

    The grammar operates on the combined alphabet:

  10.  EQUIVALENCE PROOF: NFA вҶ” RIGHT-LINEAR GRAMMAR
================================================================================

Theorem 2.
    For the Gana NFA MвӮғ with 23 states and the right-linear grammar GвӮғ
    defined in Section 7:  L(MвӮғ) = L(GвӮғ).

Proof (sketch).
    The construction is mechanical:

    (NFA вҶ’ Grammar):
    For each transition Оҙ(q, a) вҲӢ q':
        Add production q вҶ’ a q'
    For each accepting state q вҲҲ F:
        Add production q вҶ’ Оө

    Applied to MвӮғ:
        Оҙ(G1_III_0, I) = G1_III_1  вҶ’  G1_III_0 вҶ’ I G1_III_1
        Оҙ(G4_IIU_2, U) = ACCEPT    вҶ’  G4_IIU_2 вҶ’ U   (Оө for ACCEPT removed)

    The Оө-transitions (spawning) are handled by:
        SвӮҖ вҶ’ОөвҶ’ G1_III_0, G1_IIU_0
        becomes: SвӮҖ вҶ’ I G1_III_1 | I G1_IIU_1
        (compose the Оө-transition with the first real transition)

    (Grammar вҶ’ NFA):
    For each production A вҶ’ a B:
        Add transition Оҙ(A, a) вҲӢ B
    For each production A вҶ’ a:
        Add transition Оҙ(A, a) вҲӢ ACCEPT, and ACCEPT вҲҲ F

    Since the constructions are inverses, L(MвӮғ) = L(GвӮғ).  в–Ў

Corollary.
    The Ragale gana language is regular, and therefore:
    - decidable in O(n) time (linear in the number of syllables)
    - recognizable with O(1) space (constant вҖ” the NFA state set)
    - closed under intersection with other regular languages (prasa, guru)
